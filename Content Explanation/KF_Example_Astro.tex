\section*{1D Radar Tracking: The Standard vs. Extended Kalman Filter}

This technical brief isolates the exact mathematical reason why the Extended Kalman Filter (EKF) is required in Astrodynamics. We define a perfectly linear physical process (constant velocity descent), but introduce a highly non-linear measurement model (radar line-of-sight).

\subsection*{1. Analytical Problem Definition}

A spacecraft is descending vertically at a constant velocity. A ground-based radar station is located a horizontal distance $D$ away from the landing pad. The radar cannot measure altitude directly; it can only measure the line-of-sight range to the spacecraft.

\subsubsection*{The State Vector \& Linear Process Model}
The state of the spacecraft is defined by its altitude $y$ and velocity $\dot{y}$:
\begin{equation}
\mathbf{x} = \begin{bmatrix} y \\ \dot{y} \end{bmatrix}
\end{equation}

Because the velocity is constant, the physical model mapping state $k-1$ to state $k$ is perfectly linear:
\begin{equation}
\mathbf{x}_{k} = \mathbf{F} \mathbf{x}_{k-1} + \mathbf{w}_{k}
\end{equation}
\begin{equation}
\begin{bmatrix} y_k \\ \dot{y}_k \end{bmatrix} = \begin{bmatrix} 1 & \Delta t \\ 0 & 1 \end{bmatrix} \begin{bmatrix} y_{k-1} \\ \dot{y}_{k-1} \end{bmatrix} + \begin{bmatrix} 0 \\ w \end{bmatrix}
\end{equation}
where $\mathbf{w}_k$ is the process noise.

\subsubsection*{The Non-Linear Measurement Model}
The radar measures the hypotenuse of the triangle formed by the spacecraft's altitude and the radar's horizontal distance $D$. This is a non-linear relationship:
\begin{equation}
z_k = h(\mathbf{x}_k) + v_k = \sqrt{y_k^2 + D^2} + v_k
\end{equation}
where $v_k$ is the measurement noise.

\subsection*{2. The Analytical Difference: KF vs EKF}

To use a Kalman Filter, we must map the state vector to the measurement via a matrix $\mathbf{H}$. 

\vspace{0.5cm}
\noindent \textbf{The Standard KF Approach (Linearization Flaw):} \\
The standard KF requires a constant, linear $\mathbf{H}$ matrix. We linearize the radar equation via a Taylor expansion around a fixed, nominal starting altitude ($y_0$):
\begin{equation}
\mathbf{H}_{KF} = \begin{bmatrix} \frac{y_0}{\sqrt{y_0^2 + D^2}} & 0 \end{bmatrix}
\end{equation}
\textit{The Flaw:} As the spacecraft drops, the observation angle changes drastically. $\mathbf{H}_{KF}$ stubbornly assumes the initial geometry is still true, causing the filter to diverge.

\vspace{0.5cm}
\noindent \textbf{The EKF Approach (Dynamic Jacobian):} \\
The EKF recalculates the Jacobian matrix $\mathbf{H}_k$ at \textit{every single time step} using the most recent \textit{a priori} state estimate $\hat{y}_{k}^-$:
\begin{equation}
\mathbf{H}_{EKF, k} = \frac{\partial h}{\partial \mathbf{x}} \bigg|_{\hat{\mathbf{x}}_{k}^-} = \begin{bmatrix} \frac{\hat{y}_{k}^-}{\sqrt{(\hat{y}_{k}^-)^2 + D^2}} & 0 \end{bmatrix}
\end{equation}
\textit{The Fix:} The EKF physically "learns" the changing geometry. The dynamic measurement matrix reflects the new angle at each step, allowing the filter to track the true trajectory.

\subsection*{3. Python Numerical Implementation}

The following script simulates the true descent, generates noisy radar measurements, and runs both filters concurrently to demonstrate divergence vs. convergence.

\begin{lstlisting}[language=Python, caption=1D Radar Tracking: Standard KF vs EKF]
import numpy as np
import matplotlib.pyplot as plt

def simulate_radar_tracking():
    # 1. Simulation Parameters
    dt = 0.5           # Time step (seconds)
    t = np.arange(0, 30, dt)
    N = len(t)
    D = 5000.0         # Radar horizontal distance (m)
    
    # 2. True State Generation (Spacecraft dropping at 300 m/s)
    y_true = np.zeros(N)
    v_true = np.zeros(N)
    y_true[0] = 10000.0
    v_true[0] = -300.0
    
    for i in range(1, N):
        y_true[i] = y_true[i-1] + v_true[i-1] * dt
        v_true[i] = v_true[i-1]
        
    # 3. Noisy Radar Measurements (Non-linear range)
    R_var = 150.0**2  # Radar noise variance (150m standard dev)
    z = np.sqrt(y_true**2 + D**2) + np.random.normal(0, np.sqrt(R_var), N)
    
    # 4. Filter Initialization
    x_kf  = np.array([10500.0, -250.0]) 
    x_ekf = np.array([10500.0, -250.0])
    
    P_kf  = np.eye(2) * 500.0
    P_ekf = np.eye(2) * 500.0
    
    F = np.array([[1, dt], [0, 1]])
    Q = np.array([[1.0, 0], [0, 1.0]]) 
    
    # Standard KF Fixed Matrix (Linearized at starting altitude)
    y0 = x_kf[0]
    H_kf_fixed = np.array([[y0 / np.sqrt(y0**2 + D**2), 0]])
    
    y_est_kf = np.zeros(N)
    y_est_ekf = np.zeros(N)
    
    # 5. Filter Loops
    for i in range(N):
        # --- Standard KF ---
        x_kf_pred = F @ x_kf
        P_kf_pred = F @ P_kf @ F.T + Q
        z_pred_kf = H_kf_fixed @ x_kf_pred
        S_kf = H_kf_fixed @ P_kf_pred @ H_kf_fixed.T + R_var
        K_kf = P_kf_pred @ H_kf_fixed.T @ np.linalg.inv(S_kf)
        x_kf = x_kf_pred + K_kf @ (z[i] - z_pred_kf)
        P_kf = (np.eye(2) - K_kf @ H_kf_fixed) @ P_kf_pred
        y_est_kf[i] = x_kf[0]
        
        # --- Extended KF ---
        x_ekf_pred = F @ x_ekf
        P_ekf_pred = F @ P_ekf @ F.T + Q
        
        y_pred = x_ekf_pred[0]
        z_pred_ekf = np.sqrt(y_pred**2 + D**2)
        
        # EKF Dynamic Jacobian
        H_ekf_dynamic = np.array([[y_pred / np.sqrt(y_pred**2 + D**2), 0]])
        
        S_ekf = H_ekf_dynamic @ P_ekf_pred @ H_ekf_dynamic.T + R_var
        K_ekf = P_ekf_pred @ H_ekf_dynamic.T @ np.linalg.inv(S_ekf)
        x_ekf = x_ekf_pred + K_ekf @ (np.array([z[i]]) - z_pred_ekf)
        P_ekf = (np.eye(2) - K_ekf @ H_ekf_dynamic) @ P_ekf_pred
        y_est_ekf[i] = x_ekf[0]

    # 6. Plotting
    plt.figure(figsize=(10, 6))
    plt.plot(t, y_true, 'k--', linewidth=2, label='True Altitude')
    plt.plot(t, y_est_kf, 'orange', linewidth=2, label='Standard KF')
    plt.plot(t, y_est_ekf, 'green', linewidth=2, label='Extended KF')
    plt.title('KF vs EKF: Non-Linear Radar Tracking')
    plt.xlabel('Time (s)')
    plt.ylabel('Altitude (m)')
    plt.legend()
    plt.grid(True)
    plt.show()

if __name__ == '__main__':
    simulate_radar_tracking()
\end{lstlisting}
