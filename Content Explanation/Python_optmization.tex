\section*{Introduction}
In astrodynamics software like GMAT, students often confuse \textbf{Targeters} (Differential Correctors) with \textbf{Optimizers} (Gradient-Descent Solvers). This brief breaks down a simplified Python script that demonstrates the mathematical difference between finding \textit{any} valid trajectory and finding the \textit{most fuel-efficient} trajectory.

\section*{Step 1: The Simulated Environment}
Before we solve the problem, we must define the mathematical rules of our universe. In this simplified script, a spacecraft performs two maneuvers ($V_1$ and $V_2$). The final altitude achieved is a non-linear combination of these two burns, and the fuel cost is simply their sum.

\begin{lstlisting}[language=Python]
import numpy as np
from scipy.optimize import fsolve, minimize

def final_altitude(vars):
    v1, v2 = vars
    # A mock non-linear response representing trajectory propagation
    return 200 * v1 + 150 * (v2 ** 1.2) 

def fuel_cost(vars):
    v1, v2 = vars
    # Total Delta-V (fuel used)
    return v1 + v2

TARGET_ALTITUDE = 5000 # km
\end{lstlisting}

\section*{Step 2: The Targeter (Root-Finder)}
A Targeter's sole purpose is to drive an error function to exactly zero. It acts mathematically as a root-finding algorithm. 

\textbf{The Limitation:} A standard root-finder requires the system to be perfectly determined—meaning the number of variables being changed must exactly equal the number of goals. Because we only have one goal (Reach 5000 km), we can only allow the Targeter to change \textit{one} variable. If we give it two variables, the math becomes "under-determined" and the matrix inversion fails. Therefore, we must lock $V_1$ and only allow the solver to guess $V_2$.

\begin{lstlisting}[language=Python]
# We must fix Burn 1, letting the Targeter only solve for Burn 2.
fixed_v1 = 10.0

def targeter_error(v2_guess):
    # Error = Actual Altitude - Target Altitude
    return final_altitude([fixed_v1, v2_guess[0]]) - TARGET_ALTITUDE

initial_v2_guess = [10.0]

# fsolve tries to drive the error function to exactly 0
v2_solution = fsolve(targeter_error, initial_v2_guess)[0]
targeter_vars = [fixed_v1, v2_solution]
\end{lstlisting}

When run, the Targeter succeeds in hitting 5000 km, but it does so blindly. It does not evaluate if the fuel cost was efficient; it merely stops at the first valid physical solution it finds.

\section*{Step 3: The Optimizer (Gradient-Based Minimization)}
An Optimizer is fundamentally different. Instead of just finding a zero-error state, its primary goal is to minimize an \textbf{Objective Function} (in this case, \texttt{fuel\_cost}), while treating the target altitude as a \textbf{Constraint}. 

\textbf{The Benefit:} Because the Optimizer is navigating a multidimensional gradient landscape, it \textit{requires} extra degrees of freedom. We can now unlock both $V_1$ and $V_2$. The Optimizer will balance both burns simultaneously to find the absolute minimum $\Delta V$ required to hit the target constraint.

\begin{lstlisting}[language=Python]
def optimizer_constraint(vars):
    return final_altitude(vars) - TARGET_ALTITUDE

# Define the constraint: The error must equal exactly 0
constraint = {'type': 'eq', 'fun': optimizer_constraint}
initial_guess_both = [10.0, 10.0]

# Bounds to tell the optimizer we cannot have negative thrust
bnds = ((0, None), (0, None))

# Run the Optimizer (SLSQP algorithm)
optimizer_solution = minimize(fuel_cost, initial_guess_both, method='SLSQP', bounds=bnds, constraints=constraint)
\end{lstlisting}

\section*{Conclusion for Mission Design}
When students run this code, they observe two critical behaviors that translate directly to software like GMAT:
\begin{enumerate}
    \item \textbf{Targeters are rigid:} They can only answer "How do I hit the target?" and require a strict 1-to-1 ratio of variables to goals.
    \item \textbf{Optimizers are flexible:} They answer "What is the \textit{best} way to hit the target?" and utilize extra variables to minimize mass depletion or time-of-flight.
\end{enumerate}
