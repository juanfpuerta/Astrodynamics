\section*{Laboratory Objective}
In this laboratory, we will transition from using GMAT's Differential Corrector (Targeter) to the native Sequential Quadratic Programming solver (Yukon Optimizer). We will attempt a pure inclination change maneuver. You will observe firsthand how a Targeter becomes mathematically ``blind'' when given too many variables, and how the Yukon Optimizer uses an Objective Function to discover the analytically perfect, fuel-efficient solution.

% ---------------------------------------------------------
\section*{1. The Analytical Baseline}
Before opening GMAT, we must calculate the expected analytical result. We wish to change the inclination of a 300 km circular Earth parking orbit from $i = 28.5^\circ$ to $i = 45.0^\circ$. 

Because this is a pure plane change (no change to the semi-major axis or eccentricity), the burn must be executed precisely at the line of nodes, and the $\Delta V$ must be directed purely in the \textbf{Normal} direction. 

The analytical cost for a pure inclination change is:
\begin{equation}
    \Delta V = 2 V \sin \left( \frac{\Delta i}{2} \right)
\end{equation}

For a 300 km circular orbit, $V \approx 7.726 \text{ km/s}$. With $\Delta i = 16.5^\circ$:
\begin{equation}
    \Delta V = 2 (7.726) \sin(8.25^\circ) \approx 2.217 \text{ km/s}
\end{equation}

% ---------------------------------------------------------
\section*{2. The Targeter Approach (Differential Corrector)}
In the VNB (Velocity, Normal, Binormal) frame, a pure inclination change should only use the Normal vector (\texttt{Element2}). However, in the real world, errors exist. What happens if we allow the Targeter to adjust all three axes to achieve the $45^\circ$ inclination?

\begin{lstlisting}[caption=The Targeter Sequence]
Target DC1 {SolveMode = Solve, ExitMode = DiscardAndContinue};
    % We give the Targeter control over all three axes
    Vary DC1(Burn1.Element1 = 0.1, {Perturbation = 0.0001, MaxStep = 0.2}); % Velocity
    Vary DC1(Burn1.Element2 = 0.5, {Perturbation = 0.0001, MaxStep = 0.2}); % Normal
    Vary DC1(Burn1.Element3 = 0.0, {Perturbation = 0.0001, MaxStep = 0.2}); % Binormal
    
    Maneuver Burn1(Sat);
    
    % The only goal is hitting the exact inclination
    Achieve DC1(Sat.Earth.INC = 45.0, {Tolerance = 0.01});
EndTarget;
\end{lstlisting}

\begin{tcolorbox}[colback=red!5!white,colframe=red!75!black,title=The Targeter's Flaw]
If you run this in GMAT, the Targeter will likely succeed, but the total $\Delta V$ will be entirely wrong (e.g., 5.0 km/s). Why? Because a Targeter is \textbf{fuel-blind}. It found a combination of $V$, $N$, and $B$ that resulted in a $45^\circ$ inclination, but it altered the orbit's altitude and shape in the process. It solved the equation $f(x) = 0$, completely ignoring efficiency.
\end{tcolorbox}

% ---------------------------------------------------------
\section*{3. The Optimizer Approach (Yukon SQP)}
To fix this, we replace the Differential Corrector with the Yukon Optimizer. By defining an \textbf{Objective Function}, we force the algorithm to search for the combination of variables that costs the least amount of fuel. 

Recalling our Python SQP exercise, the smoothest mathematical landscape for an optimizer is a paraboloid. Therefore, our cost function will be the sum of the squares of the $\Delta V$ components.

\begin{lstlisting}[caption=The Optimizer Sequence (Yukon)]
% Create a Variable to store the cost
Create Variable FuelCost;

Optimize Yukon1 {SolveMode = Solve, ExitMode = DiscardAndContinue};
    % We give the Optimizer the exact same freedom
    Vary Yukon1(Burn1.Element1 = 0.1, {Perturbation = 0.0001, MaxStep = 0.2});
    Vary Yukon1(Burn1.Element2 = 0.5, {Perturbation = 0.0001, MaxStep = 0.2});
    Vary Yukon1(Burn1.Element3 = 0.0, {Perturbation = 0.0001, MaxStep = 0.2});
    
    Maneuver Burn1(Sat);
    
    % The Achieve command becomes a NonLinearConstraint
    NonLinearConstraint Yukon1(Sat.Earth.INC = 45.0);
    
    % Define the Objective Function (creating a mathematical 'bowl')
    FuelCost = Burn1.Element1^2 + Burn1.Element2^2 + Burn1.Element3^2;
    
    % Tell Yukon to find the bottom of the bowl
    Minimize Yukon1(FuelCost);
EndOptimize;
\end{lstlisting}

    \item What were the final $V$, $N$, and $B$ values produced by the Targeter? How much extra $\Delta V$ was wasted compared to our analytical baseline of 2.217 km/s?
    \item Look at the final $V$, $N$, and $B$ values produced by the Yukon Optimizer. Explain how the SQP algorithm mathematically realized that \texttt{Element1} and \texttt{Element3} needed to be driven to exactly $0.0$.
    \item Explain the mathematical advantage of defining \texttt{FuelCost} using squares ($E_1^2 + E_2^2 + E_3^2$) rather than an absolute magnitude ($\sqrt{E_1^2 + E_2^2 + E_3^2}$) for gradient descent algorithms.
\end{enumerate}
