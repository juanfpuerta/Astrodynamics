\section*{Laboratory Overview: Advanced Trajectories \& Estimation}
In modern space mission design, engineers must handle complex, non-impulsive maneuvers and account for uncertainties in spacecraft tracking. This laboratory builds upon basic orbital mechanics by introducing complex gravity assists, B-plane targeting, continuous Low-Thrust maneuvers, and basic state estimation using Kalman Filters.

You will utilize a hybrid computational approach: 
1. \textbf{Analytical calculations} to establish baseline physics.
2. \textbf{Python scripting} to process data and simulate basic estimation algorithms.
3. \textbf{GMAT simulations} to numerically validate mission parameters.

\textit{Important GMAT Setup Note:} Ensure your Propagator (`DefaultProp`) uses an appropriate gravity model for the flight regime. For deep space (Exercise 1), ensure Solar and Planetary point masses are active. For LEO (Exercise 2), use a suitable spherical harmonic model (e.g., JGM-2).

\section*{Exercise 1: Interplanetary Departure \& B-Plane Targeting}
\textbf{Mission Objective:} Design a preliminary trajectory for a deep-space probe traveling from Earth to Saturn, utilizing a gravitational assist from Jupiter (Earth-Jupiter-Saturn trajectory). You must determine the correct departure window and configure the 3D parking orbit geometry before executing the numerical targeting sequence.

\textbf{Given Parameters:}
\begin{itemize}
    \item Earth Parking Orbit Altitude: $h_p = 400 \text{ km}$ (Circular)
    \item Parking Orbit Inclination: $i = 28.5^\circ$ (Kennedy Space Center)
\end{itemize}

\subsection*{Part A: Python Porkchop Plot \& Departure Geometry}
\begin{enumerate}
    \item \textbf{Porkchop Plot Generation:} Using Python and SPICE kernels (or an available Lambert solver package), generate a Porkchop plot for an Earth-to-Jupiter transfer for launch windows between 2029 and 2031. 
    \item \textbf{Vector Extraction:} Identify an optimal launch date. Extract the Characteristic Energy ($C_3$) and the 3D hyperbolic excess velocity vector ($\mathbf{v}_\infty$) in the Earth Mean Equator and Equinox of J2000 frame.
    \item \textbf{Departure Angles:} Analytically calculate the Declination ($\delta$) and Right Ascension ($\beta$) of the departure asymptote based on your $\mathbf{v}_\infty$ vector.
    \item \textbf{The Inclination Trap \& RAAN:} Check if $i \ge |\delta|$. If valid, calculate the two possible Right Ascension of the Ascending Node ($\Omega$) values that will correctly align the parking orbit plane with the departure vector using the geometric constraint:
    \begin{equation}
        \sin(\Omega - \beta) = \frac{-\tan(\delta)}{\tan(i)}
    \end{equation}
\end{enumerate}

\subsection*{Part B: GMAT Simulation \& B-Plane Targeting}
In GMAT, you will set up the numerical mission. When flying by planets, targeting specific orbital elements is highly unstable. Instead, astrodynamicists use the \textbf{B-Plane} coordinates (B-dot-T and B-dot-R) to "thread the needle" during a flyby.

\begin{enumerate}
    \item \textbf{Resources Setup:} 
    \begin{itemize}
        \item Create a Spacecraft in the 400 km Earth parking orbit. Set the inclination to $28.5^\circ$ and the RAAN to one of the values calculated in Part A.
        \item Create two custom Coordinate Systems: \texttt{JupiterBPlane} (centered on Jupiter) and \texttt{SaturnBPlane} (centered on Saturn).
        \item Set up a `DifferentialCorrector`.
    \end{itemize}
    \item \textbf{Target Sequence 1: Earth Departure to Jupiter Flyby}
        \begin{itemize}
            \item Create a \texttt{Target} sequence.
            \item \texttt{Vary} the components (V, N, B) of your Earth departure burn. Use your analytical $\Delta V$ from the $C_3$ calculation as the initial guess for the velocity direction.
            \item \texttt{Maneuver} the spacecraft.
            \item \texttt{Propagate} the spacecraft to Jupiter's periapsis.
            \item \texttt{Achieve} the required B-Plane coordinates to set up the Saturn transfer: (`Sat.JupiterBPlane.BdotT` and `Sat.JupiterBPlane.BdotR`). \textit{(Note: You may estimate these values or experiment to find a flyby that slingshots you toward Saturn).}
        \end{itemize}
    \item \textbf{Target Sequence 2: Saturn Arrival}
        \begin{itemize}
            \item \texttt{Propagate} the spacecraft past Jupiter's sphere of influence.
            \item Create a second \texttt{Target} sequence.
            \item \texttt{Vary} a small Deep Space Maneuver (DSM) using a new `ImpulsiveBurn`.
            \item \texttt{Propagate} to Saturn's periapsis.
            \item \texttt{Achieve} specific B-Plane coordinates (`Sat.SaturnBPlane.BdotT` = 0, and a specific `BdotR`) to achieve a precise \textbf{polar orbit} arrival at Saturn.
        \end{itemize}
    \item \textbf{Deliverable:} Provide a screenshot of the EJS trajectory in the Solar System graphics window. Report your calculated $\delta, \beta, \Omega$ and the final `BdotT` and `BdotR` values utilized at both Jupiter and Saturn.
\end{enumerate}

\section*{Exercise 2: Low-Thrust Orbit Raising \& State Estimation}
\textbf{Mission Objective:} A next-generation satellite uses an Electric Propulsion (EP) system to continuously raise its orbit over several days. Because ground station tracking is noisy, the mission control team requires a Kalman Filter to accurately estimate the spacecraft's true altitude during the continuous burn.

\textbf{Given Parameters:}
\begin{itemize}
    \item Initial Orbit Altitude: $h = 500 \text{ km}$ (Circular, $i=0^\circ$)
    \item Spacecraft Initial Mass: $m_0 = 1000 \text{ kg}$
    \item Electric Thruster: Thrust $T = 50 \text{ mN}$, $I_{sp} = 1500 \text{ s}$
    \item Ground Radar Noise (Measurement Uncertainty): $\sigma_z = 100 \text{ m}$
\end{itemize}

\subsection*{Part A: GMAT Low-Thrust Simulation}
\begin{enumerate}
    \item \textbf{Resources Setup:} 
        \begin{itemize}
            \item Create an `ElectricThruster` and an `ElectricTank`. Attach them to the Spacecraft. 
            \item Set the thruster's thrust direction to the spacecraft's velocity vector (Local VNB, $X=1$).
            \item Create a `FiniteBurn` object and associate it with the thruster.
        \end{itemize}
    \item \textbf{Mission Sequence:}
        \begin{itemize}
            \item \texttt{BeginFiniteBurn} using the configured EP system.
            \item \texttt{Propagate} the spacecraft for 3 days.
            \item \texttt{EndFiniteBurn}.
        \end{itemize}
    \item \textbf{Deliverable:} Generate a graphical plot of Altitude vs. Time in GMAT. Report the final semi-major axis (SMA) and the total propellant mass consumed.
\end{enumerate}

\subsection*{Part B: Python Kalman Filter (Basic 1D Estimation)}
Assume during a specific 10-minute window of the EP burn, the spacecraft's altitude increases at a roughly constant velocity, but the radar measurements are extremely noisy. 
\begin{enumerate}
    \item \textbf{Data Generation:} In Python, generate an array of "True" altitudes for 600 seconds, starting at $500 \text{ km}$ and rising at a constant rate of $0.5 \text{ m/s}$. Create a "Measured" array by adding Gaussian noise ($\mu=0, \sigma=100 \text{ m}$) to the true data.
    \item \textbf{Kalman Filter Implementation:} Write a basic 1D Linear Kalman filter to estimate the altitude. 
        \begin{itemize}
            \item State vector: $x = [altitude, velocity]^T$
            \item Process Noise Covariance ($Q$): Set a low value, as we assume our kinematic model is decent.
            \item Measurement Noise Covariance ($R$): Set to the variance of the radar ($100^2$).
        \end{itemize}
    \item \textbf{Deliverable:} Plot the Noisy Measurements (scattered dots), the True Altitude (dashed line), and your Kalman Filter Estimate (solid line) on a single graph. Discuss how changing the process noise matrix $Q$ affects the "smoothness" of your filtered estimate.
\end{enumerate}
