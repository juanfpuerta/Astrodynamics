\section*{1. Analytical Solution: The DSN Measurement Model}

To process Deep Space Network (DSN) tracking data through an Extended Kalman Filter (EKF), we must rigorously define the relationship between the 3D spacecraft state and the 1D signal received at the ground station. 

Let the 2D Cartesian state vector of the spacecraft be $\mathbf{x} = [x, y, \dot{x}, \dot{y}]^T$ and the fixed ground station location be $(X_s, Y_s)$.

\subsection*{The Non-Linear Measurement Equations ($h(\mathbf{x})$)}
The DSN antenna provides two measurements: Range ($\rho$) and Range-Rate/Doppler ($\dot{\rho}$). The non-linear measurement vector $\mathbf{z}$ is defined as:
\begin{equation}
h(\mathbf{x}) = \begin{bmatrix} \rho \\ \dot{\rho} \end{bmatrix} = 
\begin{bmatrix} 
\sqrt{(x - X_s)^2 + (y - Y_s)^2} \\ 
\frac{(x - X_s)\dot{x} + (y - Y_s)\dot{y}}{\rho} 
\end{bmatrix}
\end{equation}

\subsection*{The Measurement Jacobian Matrix ($\mathbf{H}$)}
Because $h(\mathbf{x})$ is non-linear, the standard Kalman Filter will fail. The EKF requires us to calculate the Jacobian matrix $\mathbf{H}$ (the matrix of partial derivatives) evaluated at the current state estimate to locally linearize the geometry:
\begin{equation}
\mathbf{H} = \frac{\partial h}{\partial \mathbf{x}} = 
\begin{bmatrix} 
\frac{\partial \rho}{\partial x} & \frac{\partial \rho}{\partial y} & \frac{\partial \rho}{\partial \dot{x}} & \frac{\partial \rho}{\partial \dot{y}} \\[.5em]
\frac{\partial \dot{\rho}}{\partial x} & \frac{\partial \dot{\rho}}{\partial y} & \frac{\partial \dot{\rho}}{\partial \dot{x}} & \frac{\partial \dot{\rho}}{\partial \dot{y}}
\end{bmatrix}
\end{equation}

Applying calculus (quotient and chain rules) yields the analytical $\mathbf{H}$ matrix:
\begin{equation}
\mathbf{H} = 
\begin{bmatrix} 
\frac{x - X_s}{\rho} & \frac{y - Y_s}{\rho} & 0 & 0 \\[.5em]
\frac{\dot{x}}{\rho} - \frac{(x - X_s)\dot{\rho}}{\rho^2} & \frac{\dot{y}}{\rho} - \frac{(y - Y_s)\dot{\rho}}{\rho^2} & \frac{x - X_s}{\rho} & \frac{y - Y_s}{\rho}
\end{bmatrix}
\end{equation}
This exact matrix must be calculated inside the EKF algorithm at \textit{every time step} to compute the Kalman Gain.

\section*{2. Python Implementation: The EKF Update Step}

This Python snippet demonstrates how the analytical equations derived above are implemented in software. This is the exact mathematical logic running "under the hood" during the Correct step of an EKF.

\begin{lstlisting}[language=Python, caption=EKF DSN Measurement Update (Range \& Doppler)]
import numpy as np

def calculate_dsn_jacobian(state, station):
    """ Calculates the H matrix for Range and Range-Rate """
    x, y, x_dot, y_dot = state
    Xs, Ys = station
    
    # 1. Calculate non-linear Range and Range-Rate
    dx = x - Xs
    dy = y - Ys
    rho = np.sqrt(dx**2 + dy**2)
    rho_dot = (dx * x_dot + dy * y_dot) / rho
    
    # 2. Populate the Analytical Jacobian Matrix (H)
    H = np.zeros((2, 4))
    
    # Partial derivatives of Range (rho)
    H[0, 0] = dx / rho
    H[0, 1] = dy / rho
    H[0, 2] = 0.0
    H[0, 3] = 0.0
    
    # Partial derivatives of Range-Rate (rho_dot)
    H[1, 0] = (x_dot / rho) - (dx * rho_dot / rho**2)
    H[1, 1] = (y_dot / rho) - (dy * rho_dot / rho**2)
    H[1, 2] = dx / rho
    H[1, 3] = dy / rho
    
    return np.array([rho, rho_dot]), H

def ekf_update(state_pred, P_pred, z_measured, station, R_noise):
    """ Performs the Kalman Filter Correction Step """
    
    # Get expected measurements and Dynamic Jacobian
    z_expected, H = calculate_dsn_jacobian(state_pred, station)
    
    # 1. Measurement Residual (y = actual - expected)
    y = z_measured - z_expected
    
    # 2. Kalman Gain (K = P * H^T * (H * P * H^T + R)^-1 )
    S = H @ P_pred @ H.T + R_noise
    K = P_pred @ H.T @ np.linalg.inv(S)
    
    # 3. Update State and Covariance Matrix
    state_updated = state_pred + (K @ y)
    P_updated = (np.eye(4) - K @ H) @ P_pred
    
    return state_updated, P_updated
\end{lstlisting}

\section*{3. Transition to Industry: NASA GMAT Setup}

Writing custom Python scripts is excellent for learning the math. However, for real missions, we rely on validated tools like NASA GMAT. GMAT abstracts the complex Jacobian matrices and numerical integration into object-oriented configurations.

To replicate the mathematical model above in GMAT, students must configure three distinct resources: The \textbf{ErrorModel} (Measurement Noise $R$), the \textbf{TrackingFileSet} (The DSN data), and the \textbf{ExtendedKalmanFilter} itself.

\begin{tcolorbox}[colback=white,colframe=UdeAGreen,title=\textbf{GMAT Script Equivalent}]
\begin{lstlisting}[language=bash]
% 1. Define the Ground Station and Sensor Noise (The 'R' Matrix)
Create GroundStation DSN_Goldstone;
Create ErrorModel DSNRangeNoise;
DSNRangeNoise.Type = 'DSN_SeqRange';
DSNRangeNoise.NoiseSigma = 10.0; % 10 meters variance

Create ErrorModel DSNDopplerNoise;
DSNDopplerNoise.Type = 'DSN_TCP';
DSNDopplerNoise.NoiseSigma = 0.001; % 1 mm/s variance

DSN_Goldstone.ErrorModels = {DSNRangeNoise, DSNDopplerNoise};

% 2. Define the Tracking Data (The 'z' measurements)
Create TrackingFileSet DSN_Data;
DSN_Data.AddTrackingConfig = {{DSN_Goldstone, MySat, DSN_Goldstone}, 'DSN_SeqRange', 'DSN_TCP'};
DSN_Data.FileName = {'Real_DSN_Tracking_Data.gmd'};

% 3. Configure the Extended Kalman Filter (The Math Engine)
Create ExtendedKalmanFilter EKF;
EKF.Measurements = {DSN_Data};             % Ingests 'z' and 'R'
EKF.SolveFors = {MySat.CartesianState};    % Defines the 'x' vector to update
EKF.ProcessNoiseModel = DefaultProcessNoise; % The 'Q' Matrix
\end{lstlisting}
\end{tcolorbox}
